## 2. Identified Documentation Gaps

### 2.2. Interface Coverage Gaps

- **Rewind/Branch API**: The multi-turn branching logic in `AgentService.ts` lacks a formal API contract description.

### 2.3. Security Explanation Gaps
- **Audit Encryption**: While `logging_and_audit.md` discusses integrity, the lack of cryptographical signing for logs is noted as a "Residual Risk" but the specific reasoning for not implementing it (performance vs complexity) is not explicitly documented.
- **MCP Binary Validation**: The process for verifying the hash of an MCP sidecar binary before launch is missing from the Security Overview.

### 2.4. Configuration Gaps
- **`user_profile.json` Schema**: While `app_settings.json` is well-documented, the schema and mutation rules for the user identity profile in `%USERDATA%/user_profile.json` are currently missing.
