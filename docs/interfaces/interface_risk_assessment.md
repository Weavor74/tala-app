## 3. Key Safeguards

### 3.1. Quantum Firewall (PII Protection)
All outbound inference traffic and local logs pass through the **Quantum Firewall**. This service uses regex and pattern matching to detect and redact API keys, UUIDs, and PII before they cross an interface boundary.

### 3.2. Tool Blocklist
Legacy tools (e.g., `write_file`, `terminal_run`) are blocked at the execution layer in `ToolService` for all AI-driven turns, forcing use of higher-level, audited equivalents (e.g., `fs_write_text`).

### 3.3. Context Isolation
Renderer processes are denied access to the `nodeIntegration`. All communication is performed via the `contextBridge`, preventing compromised UI dependencies from accessing the host OS.

## 4. Unresolved Risks
-   **Local Inference Resource Exhaustion**: Massive context windows in Ollama can lead to CPU/RAM starvation.
-   **MCP Stability**: Child process crashes can lead to silent tool failure; partially mitigated by recovery callbacks in `AgentService`.
