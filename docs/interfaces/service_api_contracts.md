## 1. Overview
This document defines the internal "Service-to-Service" API contracts within the Electron Main process. These contracts are not exposed to the UI but govern the reliability and modularity of the core engine.

## 2. Orchestration Layer (The "Soul")
The `AgentService` acts as the primary orchestrator, coordinating between specialized services.

### 2.1. Tool Registration Contract
**Endpoint**: `ToolService.register(definition: ToolDefinition)`
-   **Input**: `name`, `description`, `parameters` (Schema), `execute` (Function).
-   **Guarantees**: The tool becomes globally available to all agent turns.
-   **Constraints**: Tool names must be unique. `LEGACY_TOOLS` (e.g., `write_file`) are blocked in favor of `fs_write_text`.

### 2.2. Memory Retrieval Contract
**Service**: `MemoryService` / `HybridMemoryManager`
-   **Methods**: `search(query, limit)`, `add(text, metadata)`.
-   **Contract**: Returns a ranked list of semantic matches with scores.
-   **Integration**: Seamlessly bridges between Mem0 (Relational) and Chroma (Vector).

## 3. Tool Execution Protocol
Tools are executed via the `executeTool` method:
1.  **Validation**: Arguments are checked against the tool's JSON Schema.
2.  **Redaction**: Arguments are redacted from audit logs if sensitive patterns are detected.
3.  **Execution**: The async `execute` function is called.
4.  **Formatting**: Results are returned as a string or a vision object (`result` + `images`).

## 4. Cross-Process Signaling
Services communicate with the Renderer via the `IpcRouter` event emitters:
-   `mainWindow.webContents.send('channel', data)`
-   **Reliability**: Services must not block the Event Loop; long-running tasks must be asynchronous or offloaded to Child Processes (MCP).

## 5. Audit & Prompt Logging
All service interactions that involve model inference are recorded by the `PromptAuditService`:
-   **Artifact**: `docs/audit/prompts/`
-   **Payload**: Includes full system prompt, history, and generated response for security forensic analysis.
