## 1. Overview
This document defines the interface for the **Model Context Protocol (MCP)** integration in Tala. MCP is used to extend the agent's capabilities via a decoupled sidecar architecture, allowing polyglot tool execution and secure resource access.

## 2. Architecture
-   **Transport**: JSON-RPC over Standard Input/Output (stdio).
-   **Lifecycle**: MCP servers are spawned as child processes by `McpService.ts`.
-   **Discovery**: Capabilities and tools are queried during the initialization handshake.

## 3. Server Inventory

### 3.1. Tala Core (`tala-core`)
**Purpose**: Primary RAG and Knowledge Retrieval tools.
| Tool | Arguments | Returns | Description |
|:---|:---|:---|:---|
| `tala_rag_search` | `query: str`, `top_k: int` | `JSON` | Searches the local vector database. |
| `tala_get_evidence` | `file_path: str` | `String` | Retrieves raw evidence from indexed files. |

### 3.2. Astro Engine (`astro-engine`)

| Tool | Arguments | Returns | Description |

| `list_agent_profiles` | `null` | `String` | Lists all registered agents. |

### 3.3. Mem0 Core (`mem0-core`)
**Purpose**: Long-term relational memory management.
| Tool | Arguments | Returns | Description |
|:---|:---|:---|:---|
| `mem0_add` | `text: str`, `user_id: str` | `String` | Stores a new fact in the memory graph. |
| `mem0_search` | `query: str`, `user_id: str` | `JSON` | Retrieves relevant memories based on semantic similarity. |

## 4. Data Exchange Format
The system uses standard FastMCP schemas for tool definitions:
```json
{
  "name": "tool_name",
  "description": "helpful description",
  "inputSchema": {
    "type": "object",
    "properties": {
      "arg1": { "type": "string" }
    },
    "required": ["arg1"]
  }
}
```

## 5. Security & Isolation
-   **Process Segregation**: Each MCP server runs in an isolated Python/Node process.
-   **Capability Gating**: Agents can only access tools explicitly assigned to their profile in `app_settings.json`.
-   **Stdout Redirection**: MCP servers must redirect all logs to `stderr` to avoid polluting the JSON-RPC stream on `stdout`.
