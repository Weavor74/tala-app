## 1. Overview
This document defines the Inter-Process Communication (IPC) boundary between the Electron **Main Process** (Trusted) and the **Renderer Process** (Untrusted UI). Communication is strictly mediated via a secure preload script and a centralized `IpcRouter`.

## 2. Architecture
The IPC system follows a **Two-Stage Isolation Pattern**:
1.  **Preload Layer**: `electron/preload.ts` exposes a limited `window.electronAPI` object using `contextBridge.exposeInMainWorld`.
2.  **Router Layer**: `electron/IpcRouter.ts` handles all incoming messages and dispatches them to specialized services (AgentService, ToolService, etc.).

## 3. Communication Pattern
-   **Request-Response**: Asynchronous `ipcRenderer.invoke` calls from the UI, returned as Promises.
-   **Events**: Unidirectional `ipcRenderer.on` listeners in the UI for streaming or status updates.

## 4. Interface Definitions

### 4.1. Agent & Chat Interface
| Channel | Type | Payload | Returns | Description |
|:---|:---|:---|:---|:---|
| `agent:chat` | Invoke | `{ message: string }` | `AgentResponse` | Main conversational entry point. |
| `agent:chat:stream` | Event | `{ chunk: string, id: string }` | N/A | Real-time token streaming to the UI. |
| `agent:session:new` | Invoke | `null` | `string` (ID) | Initializes a fresh chat session. |
| `agent:session:load` | Invoke | `{ id: string }` | `ChatMessage[]` | Retrieves history for a specific session. |

### 4.2. Configuration & Settings
| Channel | Type | Payload | Returns | Description |
|:---|:---|:---|:---|:---|
| `settings:get` | Invoke | `null` | `AppSettings` | Retrieves full system configuration. |
| `settings:set` | Invoke | `Partial<AppSettings>` | `boolean` | Updates system configuration. |
| `settings:mode:set` | Invoke | `{ mode: string }` | `boolean` | Authoritative switch for Agent Mode (RP/Hybrid/Assistant). |

### 4.3. System & Tools
| Channel | Type | Payload | Returns | Description |
|:---|:---|:---|:---|:---|
| `system:info` | Invoke | `null` | `SystemInfo` | Returns environment details (Python/Node paths). |
| `mcp:tools:list` | Invoke | `null` | `ToolSchema[]` | Lists all active MCP tools. |
| `terminal:run` | Invoke | `{ command: string }` | `ExecResult` | Executes a shell command (Sensitive). |

## 5. Security Controls
-   **Channel Whitelisting**: Only channels explicitly registered in `IpcRouter.ts` are processed.
-   **Context Isolation**: `contextIsolation: true` is enabled in the BrowserView.
-   **Input Validation**: All arguments are validated or sanitized by the `IpcRouter` before being passed to internal services.
-   **Redaction**: Sensitive keys (API keys) are redacted from logs and IPC traffic where possible.
