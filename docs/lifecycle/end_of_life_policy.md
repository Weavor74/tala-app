# End of Life Policy — Tala System

**Document ID**: TALA-LIF-EOL-001  
**Version**: 1.0.0  
**Status**: Formal  
**Owner**: Security / Compliance

## 1. Objective
Ensure the secure and orderly decommissioning of a Tala system instance.

## 2. Retirement Triggers
- Replacement by a next-generation platform.
- Security vulnerabilities identified that cannot be mitigated within the current architecture.
- Withdrawal of critical hardware/OS support.

## 3. Decommissioning Procedure

### 3.1. Data Sanitization
- All vector databases (RAG) must be deleted using DOD-compliant sanitization methods.
- The `memory/` directory and all SQLite databases must be wiped.
- Audit logs must be exported to long-term archival storage and then deleted from the host.

### 3.2. Account Termination
- REVOKE all API keys and cloud credentials (e.g., Anthropic, Gemini) associated with the instance.
- Terminate local service accounts created for background tasks.

### 3.3. Physical Archival
- Export the final Technical Data Package (TDP Indext) for historical reference.
- Securely shred any physical documentation or recovery media.
