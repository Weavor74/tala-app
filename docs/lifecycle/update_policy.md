# Update Policy — Tala System

**Document ID**: TALA-LIF-UPD-001  
**Version**: 1.0.0  
**Status**: Formal

## 1. Objective
Define the mechanisms and safety protocols for delivering updates to the Tala system.

## 2. Update Categories

### 2.1. Security Hotfixes
- **Delivery**: Immediate distribution via designated secure channels.
- **Process**: Patch applied to the latest stable branch; triggers an immediate rebuild of portable distributions.

### 2.2. Minor Updates (Feature/Bugfix)
- **Delivery**: Monthly release cycle.
- **Process**: Requires verification via the standard test suite (`npm test`, `pytest`).

### 2.3. Major Upgrades
- **Delivery**: Quarterly or Bi-annual.
- **Process**: Includes updated inference models, core protocol changes (MCP), or substantial infrastructure shifts.

## 3. Deployment Safety
1. **Staged Rollout**: Updates are first deployed to a staging/development environment.
2. **Rollback Mechanism**: As defined in `docs/build/disaster_recovery.md`, every update must support a one-command reversal of state.
3. **Data Integrity**: Automated database migrations must include a pre-check of relational/vector store integrity.
