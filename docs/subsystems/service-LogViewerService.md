## Class: `LogViewerService`

## Overview
System Diagnostics & Log Management Service.  The `LogViewerService` is responsible for aggregating, visualizing, and maintaining  the various log files generated by the Tala system.   **Supported Log Sources:** - **Audit Log**: Immutable record of agent tool calls and decisions. - **Prompt Audit**: Raw prompt/response interactions for transparency. - **Runtime Errors**: Uncaught exceptions and backend crashes. - **Performance**: Latency and resource utilization metrics.  **Core Responsibilities:** - **Persistence**: Appends records to JSONL files in the `userData/logs` directory. - **Retrieval**: Provides efficient tail-reading of logs with pagination. - **Maintenance**: Handles log clearing and structured archiving into timestamped folders. - **Health Monitoring**: Generates a system health snapshot based on recent logs. - **Correlation**: Allows filtering logs by `sessionId` or `turnId` for deep debugging.
