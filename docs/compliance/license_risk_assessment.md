## 3. Mitigation of AGPL Risk (`pyswisseph`)

### 3.1. Architectural Isolation
The `pyswisseph` library is **NOT** linked into the Tala core "private" application binary. It is isolated within a separate **MCP Sidecar Process** (`astro-engine`).

### 3.2. Boundary Enforcement
- **Communication Protocol**: Data passes between the core app and `astro-engine` via standard `stdio` (JSON-RPC). 
- **Decoupling**: The core application has no knowledge of the internal AGPL components; it only interacts with the abstract MCP interface. 
- **Compliance Status**: This "Arms-Length" interaction model protects the proprietary/private status of the Tala core logic while fulfilling AGPL obligations for the specific sidecar component.

## 4. Conclusion
The Tala system maintains a healthy license posture. The identified AGPL risk is successfully mitigated through physical process isolation and abstract protocol boundaries.
