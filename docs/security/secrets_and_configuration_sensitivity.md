## 3. Handling Procedures

### 3.1. Implementation of the "Quantum Firewall"
The recursive `redact(data)` function in `log_redact.ts` is the primary guard against accidental leakage. Any configuration block that is surfaced to the UI or written to logs is piped through this utility to remove values associated with `apiKey`, `token`, `secret`, and `password`.

### 3.2. Secret Rotation
Tala currently supports manual key rotation. Users can update their API keys via the Settings interface, which triggers an atomic `saveSettings` call via `SettingsManager.ts`. The old key is overwritten both in memory and on disk.

### 3.3. Configuration Tampering
`app_settings.json` is protected from non-authorized modification by the `SettingsManager` cache. The system re-validates the structure of the settings file upon ogni startup to ensure that malicious keys have not been injected by external processes.
