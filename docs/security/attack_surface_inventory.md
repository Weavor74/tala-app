## 3. High-Risk Exposure Points

### 3.1. Command Injection (Shell)
The most critical exposure is the `shell_run` capability. While the `CodeAccessPolicy` blocks chaining operators, an attacker could still attempt "Argument Injection" (e.g., passing `--flag="dangerous values"`) into legitimate commands.

### 3.2. Data Exfiltration via RAG
An attacker could prompt the agent to "summarize" sensitive files (e.g., `.env`) and output the result. This bypasses file-system read blocks by leveraging the agent's authorized read access. Guarded by: **Topic/PII Validators** in `GuardrailService`.
