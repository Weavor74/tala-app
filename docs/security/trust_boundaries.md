## 1. Overview
This document identifies the major trust boundaries within the Tala system where data or control privileges change. Each boundary represents a potential attack vector that is addressed by specific security controls.

## 2. Boundary Definitions

## 3. Boundary Control Details

### 3.1. The IPC Preload Bridge
The preload script (`electron/preload.ts`) acts as the "Air Gap" for the Renderer. Only the `electronAPI` object is exposed. No `require`, `process`, or `fs` modules are reachable by UI code.

### 3.2. The Tool Sandbox
The `ToolService` enforces a "Least Privilege" model. Tools like `shell_run` are only available in specific Agent Modes and are subject to the `CodeAccessPolicy`, which explicitly blocks command chaining operators (`;`, `&&`, `|`, `>`).

### 3.3. Quantization of Trust
MCP servers are treated as "Data Suppliers." They are granted read access to specific indexes (Chroma/Qdrant) but have no authority over the application's configuration or user sessions.
