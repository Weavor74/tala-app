# Verification Methods — Tala System

This document defines the taxonomy of verification methods used to validate system requirements.

## 1. Verification Hierarchy

### 1.1. Inspection (I)
Verification by visual examination of documentation, source code, or UI elements. used for requirements where a physical or logical state can be confirmed by an expert.

### 1.2. Analysis (A)
Verification using mathematical models, simulations, or logical reasoning. used for non-functional requirements like performance, security coverage, or scalability where direct testing is impractical.

### 1.3. Demonstration (D)
Verification by exhibiting a functional capability of the system under live conditions. focuses on "Can the system do X?" without exhaustive edge-case testing.

### 1.4. Test (T)
Verification via a repeatable, controlled procedure with predefined inputs and expected outputs. used for detailed functional and unit verification.

## 2. Validation Mapping

| Requirement Type | Primary Method | Supporting Evidence |
|:---|:---|:---|

| **Security** | Analysis / Test | Pentest reports, Unit tests. |
| **Performance** | Analysis / Demonstration | Benchmarking logs. |
| **Non-Functional** | Analysis / Inspection | Code coverage reports, Audit logs. |
